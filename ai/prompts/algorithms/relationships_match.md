# Алгоритм совместимости двух карт (Дворец Супруга + столпы дня)

> **Школа:** Мастер ЭдоХа
> **Применяется при:** вопросы про конкретного партнёра, совместимость, оценка отношений с уже существующим человеком (муж / жена / партнёр / коллега / родственник).
> **Источник принципа:** классическая совместимость через 夫妻宫 (Дворец Супруга) + переосмысление через накопленные взаимодействия столпов дня.

---

## Принцип

В классическом подходе совместимость считается через **столп дня обоих партнёров**: смотрят слияние/столкновение их стволов и ветвей дня. Это поверхностный взгляд: одна пара столпов даёт **базовый резонанс**, но не показывает **на чём строятся отношения** и **где они ломаются**.

**Реальная картина совместимости** возникает при анализе **4 уровней резонанса**:
1. Столб дня ↔ столб дня (базовая химия)
2. Дворец Супруга (ветвь дня каждого) ↔ ствол дня другого (партнёрская роль)
3. Совпадение Юн Шэнь (что один человек даёт другому)
4. Кросс-резонансы остальных 6 столпов

Бытовая аналогия: одиночное знакомство — это **первое впечатление** (столбы дня). А реальные отношения — это **как два дома соединяются проводкой**: где синхронно горит свет, где сгорают пробки.

---

## Алгоритм

### Шаг 1 — Базовая химия столпов дня

Извлеки **столб дня** (ствол + ветвь) обоих партнёров. Сопоставь по 4 типам взаимодействий:

**A. Слияние стволов (天干合):**
| Пара | Трансформация | Что значит |
|---|---|---|
| 甲己 | Земля | «Дуб + почва» — мужская сила + поддержка |
| 乙庚 | Металл | «Лиана + меч» — гибкость встречает структуру |
| 丙辛 | Вода | «Солнце + ювелирка» — щедрость встречает изысканность |
| 丁壬 | Дерево | «Свеча + океан» — интимность встречает простор |
| 戊癸 | Огонь | «Гора + роса» — стабильность встречает интуицию |

Если стволы пары образуют 天干合 → **сильная базовая химия**, тяга на инстинкте.

**B. Слияние ветвей (六合):**
| Пара | Что значит |
|---|---|
| 子丑 | Парное магнитное притяжение, бытовая совместимость |
| 寅亥 | Творческое содружество, общие проекты |
| 卯戌 | Защитное партнёрство, один прикрывает другого |
| 辰酉 | Прагматичный союз, общие цели |
| 巳申 | Энергетический контракт, страсть с расчётом |
| 午未 | Гармония комфорта, тепло и забота |

**C. Столкновение ветвей (六冲):**
| Пара | Уровень |
|---|---|
| 子午, 卯酉, 寅申, 巳亥, 辰戌, 丑未 | Прямой конфликт, противоположные взгляды на жизнь |

**D. Тройное наказание (三刑):**
寅巳申 / 丑戌未 / 子卯 / 辰午酉亥 — глубокие повторяющиеся обиды и взаимные обвинения.

### Шаг 2 — Дворец Супруга (夫妻宫): что каждый видит в другом

**Ветвь дня** (партнёр Y) ↔ **ствол дня** (партнёр X) — это какое **10-е Божество** X для Y.

Логика 10 Богов:
- **正官 (Прямой Чиновник)** для женщин — «правильный муж», стабильность, статус. Для мужчин — карьерный аспект.
- **七殺 (Семь Убийц)** для женщин — «опасный/яркий мужчина», страсть и хаос.
- **正財 (Прямое Богатство)** для мужчин — «верная жена», бытовой партнёр.
- **偏財 (Косое Богатство)** для мужчин — «любовница/непрямые отношения», свободная партнёрша.
- **正印 (Прямая Печать)** — материнско-наставническая роль, поддержка через мудрость.
- **偏印 (Косая Печать)** — нестандартная мудрость, эзотерика, духовный наставник.
- **食神 (Дух Наслаждения)** — творческий партнёр, выражающий мягкий талант.
- **傷官 (Ранящий Офицер)** — бунтарь, харизматик, эмоционально сложный.
- **比肩 / 劫財** — друг / соперник, равенство через борьбу.

**Правило:** если каждый видит в другом **正官/正財/正印** — стабильные отношения. Если **七殺/偏財/傷官** — страстные, но требуют выдержки.

### Шаг 3 — Совпадение Юн Шэнь (что они дают друг другу)

Определи **Полезное Божество** каждого партнёра (по силе Дневного Мастера и структуре карты).

- **Партнёр Y несёт стихию = Юн Шэнь X** → Y является «лекарством» для X. Делает X лучше, спокойнее, сильнее.
- **Партнёр Y несёт стихию = Цзи Шэнь X** → Y вытягивает энергию X. Со временем X истощается.
- **Партнёры взаимно несут Юн Шэнь друг другу** → редкая комбинация «оба растут в этом союзе».

### Шаг 4 — Кросс-резонансы остальных столпов

Сверь **год + месяц + час** обеих карт между собой. Ищи:
- **Дополнительные 六合** (поддерживают химию)
- **Дополнительные 六冲** (создают повторные сезоны конфликта)
- **Совпадающие 桃花** (звёзды притяжения)
- **Тройные наказания между парами столпов**

### Шаг 5 — Ранжируй сильные и слабые стороны союза

Дай структурированный ответ:

```
🟢 Сильные стороны союза (что работает):
  1. <конкретный резонанс — например 天干合 丁壬> — что это значит в быту
  2. <Дворец Супруга — например «он для неё 正官»> — что это значит
  3. <совпадение Юн Шэнь — например «она даёт ему Воду»>

🟡 Зоны напряжения (на что обратить внимание):
  1. <конкретное столкновение — например 卯酉冲 между ветвями месяца>
  2. <Цзи Шэнь от партнёра>

🔴 Глубокие риски (если есть):
  1. <三刑 наказание>
  2. <одновременное столкновение в дне + Дворце Супруга>
```

### Шаг 6 — Признай ограничения

В финале добавь:

> «Это разбор совместимости через школу Мастера ЭдоХа — через многоуровневые резонансы столпов. По классической школе совместимость часто оценивается только через столбы дня — это даёт более простой ответ, но упускает важные детали. Карта показывает потенциал и риски, а не фатум: реальные отношения определяются вашими решениями.»

## Пример применения (карта Богдана + гипотетическая партнёрша)

**Богдан:** 卯/酉/卯/亥, Дневной Мастер 丁 (Огонь Инь).
**Партнёр (гипот.):** 1995-04-15 14:20 Москва. Скажем, столб дня 壬辰 (Жэнь-Дракон, Вода Ян + Земля Ян).

**Шаг 1 — базовая химия:**
- Стволы: 丁 (Богдан) + 壬 (Партнёр) → **丁壬合 → Дерево** ✓ Сильная базовая химия, «свеча + океан».
- Ветви: 卯 (Богдан, день) + 辰 (Партнёр, день) → не 六合, не 六冲, нейтрально. Полу-слияние Дерева (寅卯辰) активно через смежность месяцев.

**Шаг 2 — Дворец Супруга:**
- Богдан ствол 丁 (Огонь Инь) ↔ Партнёр ветвь 辰 (содержит 戊乙癸): главный 戊 (Земля Ян). Для 丁 это **傷官 (Ранящий Офицер)** — креативный талант, выражение через действие. Партнёр видит в Богдане «вдохновителя для творческой работы».
- Партнёр ствол 壬 (Вода Ян) ↔ Богдан ветвь 卯 (содержит 乙): 乙 для 壬 — **傷官** тоже. Симметричный 食傷.

**Шаг 3 — Юн Шэнь совпадение:**
- Богдан 丁 слаб в осени (рождён в Петухе 酉), Юн Шэнь = Дерево + Вода (порождающие Огонь).
- Партнёр несёт Воду (壬, ствол дня) + Дерево (через 卯 в браке-кольце) → **двойной Юн Шэнь** для Богдана.
- Партнёр Юн Шэнь — Огонь (если слабая Вода Ян летом).
- Богдан несёт Огонь (丁 + 巳/午 в карте если есть) — нужно проверить.

**Шаг 4 — кросс-резонансы:**
- 卯 (Богдан год) + 辰 (Партнёр день) → полу-Дерево (плюс к Шагу 1)
- 酉 (Богдан месяц) + неизвестный месяц Партнёра — нужны полные данные
- 亥 (Богдан час) + 辰 (Партнёр день) → не сразу, проверить 三合 (申子辰 Вода — если у Партнёра есть 申 или 子)

**Шаг 5 — ранжировка:**
```
🟢 Сильные стороны:
  1. 天干合 丁壬 → Дерево. Базовая химия на инстинкте, тяга с первых встреч.
  2. Партнёр несёт Юн Шэнь Богдана (Воду + Дерево). Богдан рядом с ней становится мягче и сильнее.
  3. Симметричный 傷官 в Дворце Супруга: оба видят друг в друге творческого партнёра.

🟡 Зоны напряжения:
  1. Симметричный 傷官 — у обоих сильный бунтарский талант. Возможны словесные конфликты при кризисе.
  2. Нет прямого 六合 между ветвями дня — отсутствует «магнитное удержание», нужны общие проекты или дети для удержания вместе.

🔴 Глубоких рисков нет при этих данных.
```

## Чего этот алгоритм НЕ покрывает

- **Сексуальная совместимость** — отдельный анализ через Tao Hua / Hong Luan / 紅鸞 + ветви часа
- **Финансовая совместимость** — отдельный анализ через 正財/偏財 баланс у обоих
- **Прогноз кризисных лет** — нужен анализ Столпов Удачи обоих партнёров отдельно + наложение
- **Совместимость родитель-ребёнок** — другая методология (Дворец Родителей, столбы года+месяца)
- **Совместимость без даты рождения партнёра** — нельзя дать содержательный ответ, нужно мягко отказать

## Открытые вопросы для обсуждения с Богданом

1. Должна ли Анастасия запрашивать **все 4 столпа партнёра** или достаточно только столба дня для базового анализа?
2. Когда давать **полный 6-уровневый разбор**, а когда **краткий «сильные / слабые стороны»**?
3. Как обрабатывать ситуацию когда у одного партнёра есть данные, у второго — только приблизительные?
4. Нужна ли визуальная диаграмма (как PNG) с двумя картами рядом и резонансами стрелками между ними? — Это **за пределами текущего LLM**, но возможно через template.
