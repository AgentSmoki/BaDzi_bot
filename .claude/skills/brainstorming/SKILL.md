# Brainstorming

## Железное правило

**ЗАПРЕЩЕНО писать ADR или код до подтверждённого design document.**

---

## Когда использовать

Обязательно перед:
- Новым тарифом / изменением pricing (290/990/2490 RUB)
- Новым типом интерпретации Ба Цзы (новый раздел знаний)
- Изменением персонажа Анастасии (тон, стиль, ограничения)
- Новым визуальным форматом карты (Playwright шаблон)
- Интеграцией нового LLM / смена OpenRouter модели
- Архитектурными изменениями (новый сервис, замена KuzuDB)

---

## Диалог по требованиям

**Блок 1 — Пользовательский сценарий:**
- Что пользователь видит / нажимает / получает?
- Это для нового пользователя, существующего или B2B?
- Как изменится поведение по сравнению с текущим?

**Блок 2 — Монетизация (для тарифных изменений):**
- На каком тарифе доступно? (базовая / 290 / 990 / 2490)
- Базовая интерпретация ВСЕГДА бесплатна — эту инвариант нарушаем?
- 1 бесплатный вопрос для новых — остаётся?
- Что происходит при downgrade / истечении подписки?

**Блок 3 — AI и Knowledge:**
- Новый тип вопроса требует новых концептов в KuzuDB?
- Какой контекст нужен LLM (размер подграфа RAG)?
- Временные вопросы (текущий год/месяц/день) — затрагивает?
- Промпт Анастасии меняется или нет?

**Блок 4 — Технические ограничения:**
- Playwright карта нужна или только текст?
- OpenRouter — Kimi 2.6 справится или нужен fallback Claude с первого раза?
- SQLAlchemy миграция нужна?

## Итоговый Design Document

```markdown
## Фича: [название]

**Пользовательский сценарий:** [1-2 абзаца]

**Что входит в scope:**
- ...

**Что НЕ входит в scope:**
- ...

**Монетизация:**
- Тариф: [какой]
- Инвариант «базовая интерпретация бесплатна»: [нарушается / нет]

**Технические решения:**
- LLM: Kimi 2.6 / Claude fallback / другое
- KuzuDB: новые концепты [список] / без изменений
- БД: миграция нужна / нет
- Playwright: новый шаблон / существующий

**ADR которые затрагивает:** [из vision.mdc]
```

Получить явное «да» перед `writing-plans`.

## Специфичные вопросы по типичным BaDzi_bot фичам

### Новый тип интерпретации
- Это часть базовой интерпретации (бесплатно) или расширенная (платно)?
- Какие столпы/элементы участвуют? Нужны новые концепты в KuzuDB?
- Временной вопрос (удача года) или статический (характер)?

### Новый тариф
- Между какими существующими по позиции?
- Что именно открывается — количество вопросов / тип интерпретаций / карта?
- Миграция существующих подписчиков?

### Смена LLM модели
- Причина смены — качество / стоимость / availability?
- Fallback chain: новая → Kimi 2.6 → Claude? Или наоборот?
- Тест на 20-30 реальных вопросах перед production rollout?
